How Verification Works

1

Genesis Block

Each product's chain starts with a genesis block whose previous hash is all zeros. This anchors the chain.

2

Block Linking

Every subsequent block stores the SHA-256 hash of the previous block, creating an unbreakable link.

3

Hash Recomputation

During verification, each block's hash is recomputed from its data. If any field was altered, the hash will differ.

4

Tamper Detection

A mismatch in any block proves tampering. Because each block depends on the previous, changing one block invalidates all subsequent blocks.
